# Open Source Fiscal Host Readiness Checklist

This checklist helps CareerOS prepare for evaluation by a fiscal host or open source funding organization. It is not an approval guarantee.

## Project legitimacy

- [ ] Public repository contains original and meaningful work.
- [ ] Commit history demonstrates active development.
- [ ] The project has a clear public mission.
- [ ] The README explains who benefits and how.
- [ ] The MVP demonstrates a real use case.

## Open source status

- [ ] An OSI-approved license is present.
- [ ] Source code required to use the core product is public.
- [ ] Contribution guidelines are public.
- [ ] Third-party dependencies and licenses are reviewed.

## Governance

- [ ] Maintainers are identified.
- [ ] Decision process is public.
- [ ] Significant decisions use ADRs.
- [ ] Conflicts of interest can be disclosed.
- [ ] Funding does not buy project control.

## Community

- [ ] Code of Conduct is present.
- [ ] Beginner-friendly issues exist.
- [ ] Contribution areas are documented.
- [ ] Contributor recognition is planned.
- [ ] Public updates show ongoing activity.

## Financial transparency

- [ ] Funding purpose is documented.
- [ ] Expense categories are documented.
- [ ] Approval process is documented.
- [ ] Related-party transactions require disclosure.
- [ ] Impact reporting is planned.

## Security and privacy

- [ ] Security policy is public.
- [ ] Personal data risks are documented.
- [ ] Secrets are excluded from source control.
- [ ] AI data-sharing rules are documented.
- [ ] Vulnerability reporting is available.

## Evidence package

Prepare links to:

- [ ] repository;
- [ ] license;
- [ ] roadmap;
- [ ] governance;
- [ ] security policy;
- [ ] recent releases;
- [ ] active issues and pull requests;
- [ ] community channels;
- [ ] public demo or screenshots;
- [ ] project impact statement.

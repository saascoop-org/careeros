## Documentation philosophy

Each document should answer a clear question:

| Document | Question |
|---|---|
| Vision | Why does CareerOS exist? |
| Mission | What does CareerOS do now? |
| Principles | Which values guide decisions? |
| MVP | What is the smallest useful product? |
| Architecture | How is the system organized? |
| Domain | Which concepts belong to the product? |
| Roadmap | How will capabilities evolve? |
| Governance | How can people participate and decide? |
| Funding transparency | How will resources be handled openly? |
| ADRs | Why was a significant decision made? |

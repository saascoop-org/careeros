# Funding and Financial Transparency

## Purpose

Funding should support the public mission of CareerOS and reduce barriers to sustainable maintenance.

## Principles

- project funds do not purchase governance control;
- expenses should be linked to public project needs;
- funding sources and material conflicts should be disclosed;
- project resources should prioritize open source outputs;
- personal and project expenses must remain distinguishable;
- financial reporting should be understandable to contributors.

## Possible eligible expense categories

Subject to the rules of the selected fiscal host:

- hosting and infrastructure;
- domain and project services;
- accessibility work;
- security reviews;
- contributor documentation;
- translations;
- community events;
- design and usability;
- legally compliant contractor work;
- open source maintenance.

## Approval workflow

Before spending project funds:

1. document the need;
2. estimate cost;
3. identify the public benefit;
4. disclose related parties;
5. obtain the required approval;
6. publish the expense through the fiscal host;
7. link the result to the relevant issue or milestone.

## Restrictions

CareerOS should not:

- promise financial returns;
- sell influence over technical decisions;
- hide material project expenses;
- use community funds for unrelated private costs.

## Fiscal host readiness

The project should maintain:

- a public roadmap;
- an open source license;
- active development evidence;
- clear maintainers;
- contribution guidance;
- governance documentation;
- transparent expense categories;
- public impact reporting.

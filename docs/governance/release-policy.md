# Release Policy

## Versioning

CareerOS uses Semantic Versioning where practical:

- MAJOR: incompatible public contract changes;
- MINOR: backward-compatible capabilities;
- PATCH: backward-compatible fixes and documentation corrections.

During early development, contracts may evolve rapidly, but breaking changes must still be documented.

## Release requirements

A release should include:

- changelog;
- version tag;
- tests passing;
- security checks passing;
- migration notes when relevant;
- known limitations;
- updated documentation;
- artifact provenance where applicable.

## Initial releases

- `v0.1.0`: reusable Opportunity Tailoring MVP;
- `v0.2.0`: reusable professional knowledge base;
- `v0.3.0`: career intelligence capabilities.

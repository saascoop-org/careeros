# Contributor Journey

## First contribution

A new contributor should be able to:

1. understand the project in under 15 minutes;
2. run tests locally;
3. identify a beginner-friendly issue;
4. submit a focused pull request;
5. receive respectful and actionable feedback.

## Contribution areas

- Python development;
- architecture;
- data modeling;
- HR and ATS research;
- résumé templates;
- accessibility;
- security;
- documentation;
- translations;
- testing;
- design;
- community support.

## Issue labels

Recommended labels:

- `good first issue`;
- `help wanted`;
- `area:domain`;
- `area:architecture`;
- `area:docs`;
- `area:resume`;
- `area:ats`;
- `area:privacy`;
- `area:community`;
- `type:bug`;
- `type:feature`;
- `type:research`;
- `priority:now`;
- `priority:next`;
- `priority:later`.

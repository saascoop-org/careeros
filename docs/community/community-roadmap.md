# Community Roadmap

## Stage 1 — Public foundation

- complete repository documentation;
- publish MVP backlog;
- create contribution labels;
- identify small documentation and testing tasks;
- share development updates.

## Stage 2 — First external contributors

- publish good-first-issue tasks;
- add contributor recognition;
- define review expectations;
- hold public asynchronous design discussions;
- publish a monthly project update.

## Stage 3 — Domain participation

Invite participation from:

- recruiters and HR specialists;
- data and AI engineers;
- career coaches;
- accessibility specialists;
- translators;
- privacy and security contributors;
- professionals from different countries and career paths.

## Stage 4 — Shared stewardship

- add maintainers;
- establish review areas;
- document succession;
- publish funding and impact reports;
- evolve governance through a public proposal.
